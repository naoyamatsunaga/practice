# プリセット機能とホーム追加導線の実装まとめ

このドキュメントは、このトーク内で実装した内容を後から見返せるように整理したものです。

## 1. 実装した目的

- 画面構成を 4 画面にする
  - ホーム
  - プリセット管理
  - 履歴
  - 設定
- プリセット管理で CRUD（作成・一覧・編集・削除）を実現する
- プリセットに「1タップ追加 ON/OFF」を持たせる
- ホーム画面でプリセットからタスクを追加しやすくする
  - プリセット一覧から 1 件追加
  - 新規作成で追加
  - 1タップ追加 ON の項目を全件追加

---

## 2. 画面仕様（実装済み）

## 2-1. ナビゲーション（4画面構成）

`BottomNavigationBar`（`NavigationBar`）を以下の順番に変更。

1. Home
2. プリセット
3. 履歴
4. 設定

実装ファイル:
- `lib/main.dart`

## 2-2. ホーム画面

### タスク未設定時

- 一覧の代わりに `タスクを設定` ボタンを表示。
- ボタン押下で追加メニューを表示。

### タスク設定済み時

- `+`（FAB）から追加メニューを表示。

### 追加メニューの選択肢（現在は常時表示）

- プリセット一覧から追加
- 新規作成で追加
- 1タップ追加 ON の項目をすべて追加

実装ファイル:
- `lib/views/pages/home_page.dart`
- `lib/view_models/home_view_model.dart`

## 2-3. プリセット管理画面

- プリセット一覧表示
- 各行で編集/削除
- 追加ボタン（FAB）で作成ダイアログ表示

実装ファイル:
- `lib/views/pages/preset_page.dart`
- `lib/views/widgets/preset_card.dart`
- `lib/views/dialogs/add_preset_dialog.dart`
- `lib/views/dialogs/edit_preset_dialog.dart`
- `lib/views/dialogs/delete_preset_dialog.dart`

---

## 3. プリセット項目仕様

プリセットは以下の情報を保持する。

- `title`（タイトル）
- `points`（ポイント）
- `isQuickAdd`（1タップ追加 ON/OFF）
- `createdAt`
- `updatedAt`

実装ファイル:
- `lib/models/preset.dart`

---

## 4. データベース変更（Drift）

## 4-1. テーブル

- `presets` テーブルを利用
- `isQuickAdd` カラム（boolean、default false）

## 4-2. スキーマ

- `schemaVersion` を `5` に更新
- `from < 5` で `Task` / `presets` を現行定義で再作成するマイグレーションを実装

実装ファイル:
- `lib/database.dart`
- `lib/database.g.dart`（生成ファイル）

## 6. バリデーション

プリセット作成/編集ダイアログで以下をチェック。

- タイトル必須
- ポイント必須
- ポイントは数値
- ポイントは 0 以上

実装ファイル:
- `lib/views/dialogs/add_preset_dialog.dart`
- `lib/views/dialogs/edit_preset_dialog.dart`

---

## 7. 動作確認で見ておくポイント

1. アプリ起動後、下部タブが 4 つある
2. プリセット画面で作成した内容が一覧に反映される
3. 編集・削除が正しく反映される
4. ホーム画面の追加メニューから
   - プリセット1件追加
   - 新規作成追加
   - 1タップ ON 一括追加
   が実行できる
5. 一括追加時、対象が 0 件ならメッセージ表示される

---

## 8. 補足

- `database.g.dart` は `build_runner` による生成物。
- 仕様変更時は `lib/database.dart` 修正後に再生成が必要。

例:

```bash
flutter pub run build_runner build --delete-conflicting-outputs
flutter analyze
```
