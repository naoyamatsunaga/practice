# 状態管理の実装

## 概要

このアプリでは、**Riverpod**を使用して状態管理を行っています。Riverpodは、Flutterアプリケーションで状態を効率的に管理するための強力なライブラリです。

## 実装した機能

### ポイント合計値の表示

Taskの合計値をHome画面の上部に表示し、ポイントを追加・編集・削除するとリアルタイムで合計値が更新されるように実装しました。

## ファイル構成

Providerの定義は、機能ごとに以下のファイルに分かれています：

```
lib/providers/states/
├── database_provider.dart          # databaseProvider
├── activity_point_stream.dart      # activityPointsStreamProvider
└── total_points.dart               # totalPointsProvider
```

### ファイル分離のメリット

- **関心の分離**: 各Providerが独立したファイルに定義されるため、責任が明確
- **保守性**: 特定のProviderを修正する際に、該当ファイルだけを確認すれば良い
- **再利用性**: 必要なProviderだけをインポートできる
- **テスト**: 各Providerを個別にテストしやすい

## 状態管理の仕組み

### 1. Riverpodとは

Riverpodは、Flutterの状態管理ライブラリの一つです。以下の特徴があります：

- **型安全**: コンパイル時にエラーを検出できる
- **テストしやすい**: Providerを簡単にモックできる
- **パフォーマンス**: 必要な部分だけを再構築するため、効率的
- **依存関係の管理**: Provider間の依存関係を自動的に管理

### 2. Providerの種類

このアプリでは、以下の3種類のProviderを使用しています：

#### `databaseProvider` (Provider)
**ファイル**: `lib/providers/states/database_provider.dart`

```dart
final databaseProvider = Provider<AppDatabase>((ref) {
  throw UnimplementedError('databaseProviderはmain.dartで上書きする必要があります');
});
```

- **役割**: データベースインスタンスを提供
- **特徴**: アプリ全体で同じデータベースインスタンスを使用するために使用
- **上書き**: `main.dart`で実際のデータベースインスタンスで上書き
- **依存関係**: なし（最下層のProvider）

#### `taskStreamProvider` (StreamProvider)
**ファイル**: `lib/providers/states/activity_point_stream.dart`

```dart
final taskStreamProvider = StreamProvider<List<TaskModel>>((ref) {
  final repository = ref.watch(taskRepositoryProvider);
  return repository.watchTasks().map(
        (tasks) => tasks,
      );
});
```

- **役割**: Taskのリストをリアルタイムで監視
- **特徴**: データベースの変更を自動的に検知し、UIを更新
- **StreamProvider**: 非同期データ（Stream）を扱うためのProvider
- **依存関係**: `taskRepositoryProvider`に依存

#### `totalPointsProvider` (Provider)
**ファイル**: `lib/providers/states/total_points.dart`

```dart
final totalPointsProvider = Provider<int>((ref) {
  final activityPointsAsync = ref.watch(taskStreamProvider);
  
  if (activityPointsAsync.isLoading) {
    return 0;
  }
  
  if (activityPointsAsync.hasError) {
    return 0;
  }
  
  final activityPoints = activityPointsAsync.value ?? [];
  return activityPoints.fold<int>(
    0,
    (sum, activity) => sum + activity.points,
  );
});
```

- **役割**: ポイントの合計値を計算
- **特徴**: `taskStreamProvider`に依存し、リストが更新されると自動的に再計算
- **依存関係**: `taskStreamProvider`に依存

### 3. データの流れとファイル構成

```
データベース (AppDatabase)
    ↓
database_provider.dart
    ↓ databaseProvider
    ↓
task_stream.dart
    ↓ taskStreamProvider [StreamProvider]
    ↓ watchTasks() [Stream]
    ↓
total_points.dart
    ↓ totalPointsProvider [Provider] ← 合計値を計算
    ↓
Home画面 [ConsumerWidget] ← UIに表示
```

### 4. ファイル間の依存関係

```
database_provider.dart
    ↓ (インポート)
task_stream.dart
    ↓ (インポート)
total_points.dart
```

各ファイルは、必要なProviderのみをインポートしています：

- `task_stream.dart`: `task_repository.dart`をインポート
- `total_points.dart`: `task_stream.dart`をインポート
- `home.dart`: `task_stream.dart`と`total_points.dart`をインポート
- `main.dart`: `database_provider.dart`をインポート（ProviderScopeで上書き）

### 5. UIでの使用方法

#### ConsumerWidgetの使用

`Home`ウィジェットを`StatelessWidget`から`ConsumerWidget`に変更しました：

```dart
// 必要なProviderをインポート
import 'package:practice/providers/states/task_stream.dart';
import 'package:practice/providers/states/total_points.dart';

class Home extends ConsumerWidget {
  // ...
  
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    // ref.watch()でProviderを監視
    final totalPoints = ref.watch(totalPointsProvider);
    final activityPointsAsync = ref.watch(taskStreamProvider);
    
    // UIを構築
    // ...
  }
}
```

#### ref.watch()の役割

- **監視**: Providerの値を監視し、値が変更されると自動的に再構築
- **依存関係の構築**: Provider間の依存関係を自動的に構築

### 6. リアルタイム更新の仕組み

1. **データベースの変更**: ポイントを追加・編集・削除すると、データベースが更新される
2. **Streamの通知**: `watchTasks()`が新しいデータを通知
3. **StreamProviderの更新**: `taskStreamProvider`が新しいリストを受け取る
4. **依存Providerの再計算**: `totalPointsProvider`が自動的に再計算される
5. **UIの更新**: `ref.watch()`で監視しているWidgetが自動的に再構築される

### 7. main.dartでの設定

```dart
import 'package:practice/providers/states/database_provider.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  final database = AppDatabase();
  runApp(
    ProviderScope(
      overrides: [
        // databaseProviderを実際のデータベースインスタンスで上書き
        databaseProvider.overrideWithValue(database),
      ],
      child: MyApp(database: database),
    ),
  );
}
```

- **ProviderScope**: アプリ全体でRiverpodを使用できるようにする
- **overrides**: 既存のProviderを上書きする（この場合、データベースインスタンスを提供）
- **インポート**: `database_provider.dart`をインポートして`databaseProvider`にアクセス

## メリット

1. **自動更新**: データが変更されると、関連するUIが自動的に更新される
2. **型安全**: コンパイル時にエラーを検出できる
3. **テストしやすい**: Providerを簡単にモックできる
4. **パフォーマンス**: 必要な部分だけを再構築するため、効率的
5. **保守性**: 各Providerが独立したファイルに分離されているため、修正が容易
6. **可読性**: ファイル構成が明確で、どのProviderがどこにあるかが分かりやすい

## まとめ
